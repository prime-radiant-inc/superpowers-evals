# B1 root cause: where claude 2.1.177 writes its transcript (reproduction)

Date: 2026-06-13. Method: systematic-debugging reproduction (Task 7 of the ATIF
foundation plan). **This corrects the earlier B1 hypothesis.**

## Experiment

```
claude --version            # 2.1.177
TMP=$(mktemp -d); CFG=$TMP/cfg; WD=$TMP/wd
( cd "$WD" && CLAUDE_CONFIG_DIR="$CFG" claude -p "create hello.txt containing hi, then stop" )
find "$CFG" -type f
```

## Result — the "moved to sessions/" hypothesis is REFUTED

claude 2.1.177 wrote a **12.5 KB transcript to the LEGACY layout**:

```
<CFG>/projects/<munged-cwd>/<uuid>.jsonl
```

- **No `sessions/` directory was created.** The earlier-observed
  `~/.claude/sessions/<uuid>/history.jsonl` files are empty/stale (dated
  2025-12) — not the live transcript sink.
- No leakage to the real `~/.claude` (jsonl count before == after).
- This is exactly the path `coding-agents/claude.yaml` globs
  (`${CLAUDE_CONFIG_DIR}/projects` + `**/*.jsonl`).

**So 2.1.177 writes its transcript exactly where the harness looks.** B1 is NOT
a claude transcript-location change.

## Where B1 actually lives

In the failing quorum smoke run
(`results/00-quorum-smoke-hello-world-claude-20260613T175406Z-cdcb/`), the
isolated config had `projects/<munged-workdir>/memory/` but **no `*.jsonl`** —
claude wrote no transcript. Yet a direct `claude -p` with an isolated
`CLAUDE_CONFIG_DIR` *does* write the jsonl. The difference is therefore in **how
quorum/gauntlet launches claude**, not in claude's behavior. Candidate causes
for the follow-up investigation:
- gauntlet terminates the claude process before it flushes the session file;
- a launch flag / invocation mode (interactive vs `-p`) under which 2.1.177
  doesn't persist a rollout;
- the launcher's cwd/config handling producing a project dir without a session.

**Consequence:** the **2.1.175 pin may not actually fix B1** if the cause is
quorum's launch path rather than the claude version. The pin should be validated
with a real `quorum run --coding-agent claude` (it was assumed, not verified).
Re-open B1 as a quorum-claude-launch bug; demote the audit-log claim that the
root cause is "CONFIRMED: moved to sessions/<uuid>/history.jsonl" — that is
wrong.
