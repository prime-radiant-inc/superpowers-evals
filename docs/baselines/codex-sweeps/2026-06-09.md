**Cross-cutting signature — "loads the gate, then disobeys it."** Codex's two
distinctive fails (`triggering-writing-plans`, `worktree-caller-consent-gate`)
are the same shape: codex *engages* the skill ecosystem — it loads multiple
SKILL.md files, and in the worktree case it loads the very skill that holds the
Step-0 consent gate — then **overrides the mandatory pre-implementation gate and
barrels into code anyway** (writes tests / edits source on `main`). This is the
mirror image of kimi, which *under-invokes* the gating skill
([`../kimi-sweeps/2026-06-09.md`](../kimi-sweeps/2026-06-09.md)).

**Variance, not a permanent gap.** Both gate scenarios are sentinel scenarios
codex has passed before on the identical `gpt-5.5` model (verified: 06-02 and
06-05). The 06-09 fails are run-to-run behavioral variance — exactly what these
sentinels exist to catch — not a regression in codex or the harness.

eager-brainstorming cluster (action item #2, owned by Matt in
[`../2026-06-09.md`](../2026-06-09.md)), now seen on a fourth backend with the
same mechanism.
