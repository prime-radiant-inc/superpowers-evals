# 2026-06-23: GLM-5.2 full-suite benchmark (32×) + failure root-cause

**Goal**: run the **entire** scenario suite against GLM-5.2 across the three
GLM-capable harnesses (pi, opencode, codex) at high concurrency — both to
manufacture load on a new GLM dev endpoint and to characterize how GLM-5.2
performs on the superpowers behavioral suite. Follow-on to
[`2026-06-22-glm-5.2-benchmark.md`](2026-06-22-glm-5.2-benchmark.md) (which
motivated the credential axis); this run used the landed credential axis.

## Failure root-cause (55 non-passing cells, 1 subagent each)

A 55-agent workflow (one per non-passing cell; ~2.4M tokens) read each
`verdict.json` + `trajectory.json` (+ gauntlet log for indeterminates) and
classified. Result: **27 expected behavioral mismatches** (GLM doesn't
reproduce the Claude-specific skill-triggering the ACs grade — not bugs) and
**28 actionable**.

|---|---|---|---|---|
| expected_behavioral_mismatch | 27 | | none-expected | 21 |

### GLM-5.2 model characteristics (real model signal)
- **Unreliable tool-call JSON** (8 cells): missing quotes (`"agent_type": defaultt"`),
  hallucinated tool names (`multi_agent_v1`), XML fragments in the
  function-name field, omitted required params (`task`). Worst on the
  codex/responses subagent-dispatch calls.
- **Stalls — reasoning without acting**: emits `reasoning_content` then a
  text-only message with no `tool_calls`; session hangs
  (`cost-spec-plan-duplication`, `worktree-detached-head-external`).
- **Slow inference**: ~60s/response gaps + long CoT exhausted the Gauntlet
  time budget mid-task on the heavy SDD scenarios (`sdd-svelte-todo` ran 94m).
- **Tool-result bloat**: reads all files in full instead of grep
  (`cost-tool-result-bloat`, both opencode + codex) — the exact anti-pattern
  that scenario measures.

### Codex `spawn_agent` — NOT a harness flag bug (reclassified)
9 of 10 `responses_api_issue` cells are codex SDD/subagent scenarios that hit
`"unsupported call: spawn_agent"`. The root-cause subagents guessed a missing
`multi_agent = true` in the codex config. **That is wrong** — verified with
`codex features list` under the harness's exact `config.toml`: `multi_agent` is
**stable and defaults to `true`** (still `true` with our `[features]` block).
The real cause is GLM-5.2 emitting malformed/hallucinated `spawn_agent` tool
calls that codex rejects, compounded by codex's spawn_agent exposure over a
custom Responses-API provider. This is a model/interaction issue, not a
config-flag fix; making codex SDD work with GLM needs deeper investigation.

### Harness bugs found (and their fixes — this commit series)
1. **opencode capture timeout too short under load** — `opencode session list`
   (the pre-run snapshot) timed out at 30s under 32× contention →
   setup-phase indeterminate (`worktree-skill-invocation-is-consent`). Fix:
   raise `OPENCODE_CAPTURE_TIMEOUT_MS` to 90s.
2. **No scenario-level OS eligibility** — `codex-windows-session-start-hook`
   is Windows-only (PowerShell AC) but the matrix keys OS off the *agent*, so
   it ran on Linux and went indeterminate. Fix: scenario `# os:` directive the
   matrix honors (skip cells whose run OS the scenario doesn't support).
3. **Playwright post-check is environment-fragile** — `sdd-svelte-todo[-opus48]`
   post-check `npx --no-install playwright test` requires a pre-installed
   Chromium; on hosts without it the check errors → indeterminate. Fix: ensure
   the browser is installed in the check.

## Verdict
- GLM-5.2 ≈ 56% on the Claude-tuned suite; genuine misses are tool-call-JSON
  reliability, stalling, slow inference, and read-everything bloat.
- The GLM endpoint sustained 32× concurrent with no infra errors.
- 3 harness bugs fixed; codex SDD-with-GLM remains an open model/interaction
  problem (not a quick fix).

## Negative result / caveat
With gauntlet now defaulting to the subscription OAuth token when present, a
high-concurrency `run-all` drives N gauntlet drivers on the **subscription**,
whose interactive-sized rate caps will throttle heavy batches. For load runs,
prefer `ANTHROPIC_API_KEY` (unset `CLAUDE_CODE_OAUTH_TOKEN`) or lower `--jobs`.
