# Phase I — port setup_helpers (fixture creators) to TS Implementation Plan

> Independent leaf phase — can run ANY time (no dep on the runner/CLI port; `setup.sh` invokes it by string). Lowest priority / deferrable: the Python `setup-helpers` CLI can stay indefinitely since scenarios call it via a process boundary. Port when convenient or when retiring the last Python.

**Goal:** Port `setup_helpers/` (25 files, 39 registry entries — fixture creators) + the `setup-helpers run <helper>` CLI from Python to TS/bun, parity-locked by the setup-helper tests.

**Architecture:** Strangler-fig, build-ahead, then flip `setup.sh`'s `setup-helpers` invocation to the bun CLI. These helpers build scenario fixtures (git repos, plan scaffolds, planted-bug repos, etc.) under `$QUORUM_WORKDIR`, receiving `template_dir`/`superpowers_root` by signature introspection.

## Scope
- `setup_helpers/__init__.py` `HELPER_REGISTRY` (39 entries) → a TS registry mapping name → creator fn.
- `setup_helpers/base.py` (141) — shared git/fixture primitives (`create_base_repo`, git config, env) — port first (most helpers build on it).
- The ~22 helper modules (sdd plans, cost fixtures, worktree, code-review/planted-bugs, spec-* , triggering-*, claim/phantom/review-pushback, etc.) — each a small fixture creator.
- `setup_helpers/cli.py` — `setup-helpers run <helper> [...]`: look up in the registry, fill `template_dir` (`$QUORUM_REPO_ROOT/fixtures/template-repo`) + `superpowers_root` (`$SUPERPOWERS_ROOT`) by signature introspection, invoke against `$QUORUM_WORKDIR`. In TS, replicate the "first param is `workdir`; optional `template_dir`/`superpowers_root` params filled from env" convention (explicit per-helper param declaration instead of Python `inspect`).

## Tasks (subagent-driven; parallelizable)
- [ ] **Task 1:** port `base.py` primitives + tests (`create_base_repo`, git setup). Most helpers depend on these.
- [ ] **Tasks 2..N:** port the helper modules in small batches (group related: the `sdd_*`, the `cost_*`, the `spec_*`, the worktree/review/verification ones), each with its ported tests (`tests/quorum/test_cost_setup_helpers.py`, `test_sdd_*`, `tests/setup_helpers/*`, etc. — map test → helper).
- [ ] **Final task:** the `setup-helpers` bun CLI + registry; then flip `setup.sh` callers (`uv run setup-helpers run <h>` → the bun equivalent), parity-verified (run a sample scenario's setup.sh, confirm the fixture is byte-identical), and retire the Python `setup_helpers/`.

## Notes
- Helpers must produce **byte-identical fixtures** to the Python (git trees, file contents) — a `quorum run` setup depends on them; parity-check by diffing a built fixture dir Python-vs-TS.
- Per the eval CLAUDE.md: skill-execution fixture plans should be generated by the skill under test, not hand-written — preserve that property (these helpers scaffold the *non-plan* parts).
- This is a large, mechanical batch; lowest priority. Defer until the analysis/runner/CLI phases settle, or run opportunistically in parallel since it's independent.
